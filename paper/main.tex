\begin{abstract}
This paper presents an empirical performance evaluation of continuous bimanual hand tracking integrated into a Unity 2D arcade game using the homuler MediaPipe Unity Plugin (v0.16.3). The left hand controls locomotion and jumping through finger-extension states, while the right hand controls aiming and shooting. We benchmark FPS, pipeline latency, and CPU/GPU utilization across four scenarios: keyboard baseline (S1), single-hand (S2), dual-hand (S3), and stress conditions (S4). \section{Introduction}
Natural User Interfaces (NUI) enable camera-based game control without specialized hardware. MediaPipe provides efficient on-device hand landmark estimation, and the homuler MediaPipe Unity Plugin bridges native C++ inference with Unity's C\# runtime. However, continuous dual-hand tracking inside a frame-rate-sensitive 2D game loop increases main-thread workload through texture transfer, native inference, and managed result marshaling.

Most prior work focuses on discrete gesture triggers. Our project implements \textit{continuous} bimanual control: finger counts on the left hand drive horizontal movement and jumping, while the right-hand index tip and grip gesture drive aiming and shooting. This paper measures the FPS/latency trade-off of that design on consumer laptop hardware.

Contributions: (1)~a deployed bimanual control framework for 2D gameplay; (2)~a reproducible four-scenario benchmark with CSV logging via \texttt{PerformanceBenchmark}; (3)~empirical analysis of MediaPipe overhead in Unity Editor Play Mode.

\section{Related Work}
Computer vision has been integrated into games via OpenCV, depth sensors, and lightweight neural networks. MediaPipe supports real-time hand landmark estimation~\cite{b1}, and the homuler plugin exposes Hand Landmarker to Unity~\cite{b2}. Published performance data for continuous dual-hand gameplay in Unity remains limited; this work fills that gap with scenario-based measurements.

\section{System Architecture and Implementation}
The pipeline has three stages: webcam acquisition, native MediaPipe inference, and managed coordinate mapping with game input dispatch.

\subsection{System Overview and Data Pipeline Flow}
A webcam captures frames at 30\,fps. \texttt{WebCamSource} feeds \texttt{HandLandmarkerRunner}, which converts RGBA32 textures, applies flip/rotation, and invokes the native \texttt{HandLandmarker} task (\texttt{numHands = 2}). Landmarks are stored in a locked \texttt{HandLandmarkerResult} buffer and consumed by \texttt{HandInputProvider}; \texttt{PlayerController} drives \texttt{Rigidbody2D} and \texttt{GunController}.

\subsection{Continuous Coordinate Mapping and Bimanual Control Logic}
\textbf{Screen mapping:} Landmark~8 maps to screen space via $X_{screen}=x\cdot W_{screen}$, $Y_{screen}=(1-y)\cdot H_{screen}$, then \texttt{Camera.ScreenToWorldPoint}.

\textbf{Bimanual logic:}
\begin{itemize}
\item \textbf{Left hand:} One extended finger $\rightarrow$ move left; two or more $\rightarrow$ move right; four fingers $\rightarrow$ jump.
\item \textbf{Right hand:} Landmark~8 defines aim; Euclidean distance between Landmarks~4 and~5 below 0.06 triggers shooting.
\end{itemize}
If no hand is detected, keyboard/mouse input is used.

\subsection{Frame Rate Stability}
Keyboard baseline (Fig.~\ref{fig:scenarios}a) stays near 60 FPS without MediaPipe. Dual-hand tracking (Fig.~\ref{fig:scenarios}d--g) adds 7--15\,\% frame-time overhead from \texttt{CPUAsync} readback and native inference. Combat scenes (Fig.~\ref{fig:scenarios}g) show the lowest FPS ($\approx$42.6) due to combined game logic and vision workload.

\subsection{Latency and Feasibility}
On the test machine (i7-7600U, 12\,GB RAM, Intel HD Graphics 620), single- and dual-hand modes remained responsive for casual 2D gameplay under normal lighting. Without a discrete GPU, CPU-side \texttt{CPUAsync} transfer and inference dominate overhead. S4 shows increased jitter when tracking confidence drops. Evaluating GPU delegates on discrete-GPU hardware and applying Kalman filtering for smoothing are promising next steps.

\section{Conclusion and Future Work}
We implemented and evaluated continuous bimanual MediaPipe hand tracking in a Unity 2D game. Scenario screenshots (Fig.~\ref{fig:scenarios}) demonstrate gesture-to-action mapping from baseline keyboard input through dual-hand combat. Benchmarks quantify FPS/latency trade-offs on ultrabook-class hardware without discrete graphics. Future work includes standalone build profiling, GPU-delegate comparison on discrete-GPU systems, and jitter reduction under low light.
